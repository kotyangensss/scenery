## Задание 2

### Хранение данных для поиска O(1)

Используем хэш-таблицу (HashMap). Рассчитаем объем памяти для хранения номеров телефонов и полных имен.

Номер телефона хранится в виде строки длиной 15 символов максимально, каждый символ занимает 2 байта плюс 36 байт, которые требует сама java для дополнительной информации:
Память  на  один  номер  телефона=15×2+36=66 байт
Память  на  одно  полное  имя=20×2+36=76 байт

 Накладные расходы на хэш-таблицу:

-   Ссылки на `Entry` объект: 32 байта.
-   Накладные расходы: 8 байт.

Итого, для каждого элемента в HashMap потребуется:

Общий объем памяти на элемент=66 + 76 + 32 + 8 = 182 байта

Учитывая 18 758 328 уникальных записей:

Общий объем памяти = 18758328 × 182 ≈ 3.42 ГБ

### Оценка минимального использования памяти

Для минимального использования памяти можно использовать массив строк.

 Каждая ссылка занимает 4 байта в массиве.

Общий объем памяти на элемент=66 + 76 + 4 = 146 байтов

Общий объем памяти=18758328 × 146 ≈ 2.55 ГБ
